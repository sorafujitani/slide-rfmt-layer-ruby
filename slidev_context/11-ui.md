# User Interface in Slidev

## Overview

Slidev provides a rich user interface with navigation controls, presenter mode, and various interactive features.

## Navigation Bar

### Location
- Appears in the **bottom left corner** when in Play mode
- Can be extended via Global Layers

### Features
- Slide navigation
- Fullscreen toggle
- Dark mode toggle
- Quick overview access
- Drawing tools
- Recording controls

## Keyboard Shortcuts

### Navigation

| Shortcut | Action |
|----------|--------|
| `→` / `Space` | Next animation or slide |
| `←` | Previous animation or slide |
| `↑` | Previous slide |
| `↓` | Next slide |
| `Home` | First slide |
| `End` | Last slide |

### Display Controls

| Shortcut | Action |
|----------|--------|
| `f` | Toggle fullscreen |
| `o` | Toggle Quick Overview |
| `d` | Toggle dark mode |
| `g` | Show goto dialog |

### Presenter Mode

| Shortcut | Action |
|----------|--------|
| `p` | Open presenter mode |

### Drawing & Recording

| Shortcut | Action |
|----------|--------|
| `c` | Toggle drawing mode |
| `r` | Start/stop recording |

### Other

| Shortcut | Action |
|----------|--------|
| `?` | Show keyboard shortcuts |
| `Escape` | Cancel/close dialogs |

## Quick Overview

### Accessing Overview
- Press `o` keyboard shortcut
- Click overview button in navigation panel

### Features
- Grid view of all slides
- Thumbnail previews
- Quick navigation by clicking slides
- Keyboard navigation in overview mode

### Use Cases
- Quickly jump to specific slides
- Get overview of presentation structure
- Navigate non-linearly during Q&A

## Presenter Mode

### Accessing Presenter Mode

**Method 1**: Click button in navigation panel

**Method 2**: Visit URL directly
```
http://localhost:3030/presenter
```

**Method 3**: Press `p` keyboard shortcut

### Recommended Setup

Open **two browser windows**:
1. **Audience window**: Play mode (fullscreen)
   ```
   http://localhost:3030/
   ```

2. **Presenter window**: Presenter mode
   ```
   http://localhost:3030/presenter
   ```

### Features

- **Current slide**: Large view of current slide
- **Next slide preview**: See upcoming slide
- **Presenter notes**: View slide notes
- **Timer**: Track presentation time
- **Click counter**: See current click position
- **Synchronized navigation**: All windows stay in sync

### Navigation Sync

All opened pages (audience, presenter, overview) are synchronized:
- Advancing slides in presenter mode updates audience view
- Navigation in any window updates all others
- Perfect for multi-screen presentations

## Slide Overview

### Availability
Available since **v0.48.0**

### Accessing Overview
- Via Quick Overview (`o`)
- Direct URL: `http://localhost:3030/overview`

### Features

1. **Linear slide list** with notes
2. **Note editing**: Edit notes directly in overview
3. **Step preview**: Preview each click step
4. **Search**: Find specific slides
5. **Filtering**: Filter by tags or content

### Use Cases
- Review all presenter notes
- Edit notes without switching slides
- Print presenter notes
- Prepare for presentation

## Global Layers

### Purpose
Extend UI with custom elements that appear across all slides.

### Creating Global Layers

Create `global-top.vue` or `global-bottom.vue`:

```vue
<!-- global-top.vue -->
<template>
  <div class="fixed top-0 left-0 right-0 p-4 bg-primary text-white">
    Custom header
  </div>
</template>
```

### Layer Types
- `global-top`: Above slide content
- `global-bottom`: Below slide content

## Customizing UI

### Hide Navigation

```md
---
hideInToc: true
---

# Slide without navigation
```

### Custom Controls

Add custom UI elements via global layers:

```vue
<!-- global-bottom.vue -->
<template>
  <nav class="fixed bottom-4 right-4">
    <button @click="customAction">Custom Button</button>
  </nav>
</template>

<script setup>
function customAction() {
  // Custom functionality
}
</script>
```

- Focus indicators

### Best Practices
- Use semantic HTML
- Add alt text to images
- Ensure sufficient color contrast
- Test with screen readers

## URL Parameters

Control presentation via URL:

```
# Start at specific slide
http://localhost:3030/5

# Start in presenter mode
http://localhost:3030/presenter

# Start in overview
http://localhost:3030/overview

# Start with specific click
http://localhost:3030/5?clicks=3
```

## Best Practices

1. **Learn shortcuts**: Memorize key keyboard shortcuts
2. **Use presenter mode**: Essential for professional presentations
3. **Practice navigation**: Get comfortable with UI before presenting
4. **Test on target device**: Ensure UI works on presentation hardware
5. **Customize minimally**: Don't over-customize standard UI
6. **Provide fallbacks**: Have backup navigation methods
