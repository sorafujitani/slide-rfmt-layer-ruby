指定されたコンテンツMarkdownファイルを読み込んで、Slidev形式のスライドを自動生成してください。

## 実行手順

1. **コンテンツMDの読み込み**
   - ファイルパスが指定された場合: そのファイルを読み込む
   - ファイルパスが指定されていない場合: デフォルトで`note.md`を読み込む
   - ファイルが存在しない場合はエラーメッセージを表示

2. **`workflows/slide-generation.md`の参照**
   - スライド生成ワークフローを読み込む
   - 変換ルールとロジックを理解する

3. **`design_system/ai-guide.md`の参照**
   - スライドテンプレートとコンポーネント選択ルールを確認
   - 既存のコンポーネントを活用する

4. **構造分析と変換**
   - frontmatterから表紙情報を抽出
   - 見出しレベルを解析（H1, H2, H3）
   - 各セクションの内容を分類（テキスト、リスト、コード、画像）
   - スライド単位に分割（7±2の法則）

5. **コンポーネント選択と生成**
   - 表紙: `CoverSlide`
   - セクション区切り: `layout: center` + `slide-gradient-bg`
   - コード + 説明: `TwoColumnLayout`
   - 画像: `CenteredImage`
   - その他: デフォルトレイアウト

6. **slides.mdの生成**
   - Slidev形式のMarkdownを生成
   - ユーザーに生成内容を確認してもらう
   - 確認後、`slides.md`に書き込む（または追加）

7. **視覚的確認**
   - `chrome-devtools-mcp`でブラウザ表示を確認
   - レイアウトや見た目に問題がないかチェック

## 重要な注意事項

- **既存のコンポーネントを優先**: 手動HTMLを書かない
- **1スライド = 1メッセージ**: 情報を適切に分割
- **一貫性を保つ**: 同じタイプの内容には同じレイアウト
- **デフォルト設定を重複記述しない**: `transition`などは不要

## 使用例

デフォルト（`note.md`を使用）:
```
/gen-slide
```

ファイルを指定:
```
/gen-slide content/example.md
```

または

```
/gen-slide content/my-presentation.md
```
